Hospital SQL Analysis Project

 Project Overview

This project analyzes hospital patient data to identify **waiting time issues, peak hours, and doctor workload** using SQL.
The goal is to improve hospital efficiency and patient experience through data-driven insights.

 Business Problem

Hospitals often face

  Long patient waiting times
  Uneven doctor workload
  Peak hour congestion

 This project solves these problems by analyzing hospital data and finding actionable insights
# Dataset Information

The dataset contains:
 * Patient ID
 * Visit Date & Time
 * Department
 * Doctor Name
 * Waiting Time
 * Treatment Details

# Tools & Technologies Used
 * SQL
 * PostgreSQL (pgAdmin 4)

# Analysis Performed
 * Calculated patient waiting time using SQL
 * Identified peak hours
 * Analyzed doctor workload
 * Department-wise performance analysis

# Key Insights

 *  Peak hours: **10 AM – 11 AM**
 *  Some doctors are **overloaded**
 *  Cardiology department is the **busiest**

# Solutions / Recommendations
 * Smart doctor scheduling
 * Improve appointment system
 * Distribute patient load evenly
 * Add more staff during peak hours

# Conclusion

This project demonstrates how SQL can be used to solve real-world business problems in healthcare by generating meaningful insights from raw data.
